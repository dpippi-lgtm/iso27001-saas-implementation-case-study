# ISO 27001 SaaS Implementation Case Study

##  Overview
End-to-end implementation of an ISO/IEC 27001 Information Security Management System (ISMS) for a fictional SaaS company (CloudCore).

This project demonstrates practical ISMS design, risk management, control selection, and audit readiness.

---

##  Context
- SaaS (B2B platform)  
- 50 employees  
- AWS-based infrastructure  
- Customer business data in scope  

---

##  Objective
Design and implement a risk-based ISMS aligned with ISO 27001 and prepare the organization for certification readiness.

---

##  Approach

- Defined ISMS scope and boundaries  
- Conducted risk assessment (Likelihood × Impact)  
- Mapped risks to Annex A controls  
- Implemented core security controls (MFA, RBAC, logging)  
- Developed lightweight, usable policies  
- Completed internal audit and remediation  

---

##  Key Deliverables
- Risk Register  
- Statement of Applicability (SoA)  
- Security Policies  
- Implementation Roadmap  
- Audit Readiness Checklist  

---

##  Key Decisions
- Prioritized MFA and access control first  
- Kept policies minimal and actionable  
- Limited scope for faster certification readiness  

---

##  Outcome
- Reduced high-risk exposure areas  
- Implemented core ISMS controls  
- Achieved audit-ready security posture  

---

##  Focus Areas
- Risk-based security design  
- Practical ISO 27001 implementation  
- Real-world trade-off decisions
